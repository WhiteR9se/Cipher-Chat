CipherChat

Secure, end-to-end encrypted group chat written entirely in C++.

Why It Matters

Network Engineering

Implements raw TCP sockets, concurrency, framed protocols, and room routing without relying on external frameworks.

Modern Cryptography

Combines X25519 Diffie–Hellman, HKDF-SHA256, and AES-256-GCM to provide authenticated, end-to-end encrypted messaging.

Demonstrable Security

Server logs and packet captures show only ciphertext; clients alone render plaintext, proving the E2EE story in minutes.

System Overview

The relay (`./build/server`) only brokers presence and room membership. Terminal clients (`./build/client`) handle key exchange, encryption, and UI. Each room keeps a rolling history of the last 100 ciphertext envelopes for late joiners; room logs are deleted automatically when the room empties.

Layer Technique Notes
Handshake X25519 Diffie–Hellman Fresh key pair per client connection.
Key Derivation HKDF-SHA256 (32 bytes) Context-specific labels for control vs. chat channels.
Control Frames AES-256-GCM AAD binds to the client ID; used for room keys and events.
Chat Frames AES-256-GCM AAD binds to room ID & sender ID; server forwards ciphertext only.

Build & Run (Linux)

sudo apt install build-essential cmake libssl-dev
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build

# Launch relay on port 7777
./build/server 7777

# Connect clients (new terminal per user)
./build/client 127.0.0.1 7777 alice
./build/client 127.0.0.1 7777 bob

Using the Client

  • /rooms – list rooms with numeric IDs.
  • /join <name> – create or join a named room (auto key rotation).
  • /join <id> – join by numeric ID.
  • /leave – exit the current room.
  • /rename <alias> – change your display name.
  • /quit – cleanly disconnect.

Proving End-to-End Encryption

  • Server log: logs/server.log retains timestamped handshake, join/leave, and ciphertext summaries across restarts.
  • Room logs: logs/room_<id>.log contain Base64 ciphertext while rooms are active; they disappear automatically when the room empties or the relay shuts down.
  • Packet capture: tcpdump -X -s0 port 7777 shows non-human-readable payloads only.
  • Client UI: the only place plaintext appears; all other layers remain gibberish.

Demo Script (2 Minutes)

  1. Start the relay and two clients; highlight the handshake entries in server.log.
  2. Share messages and show the corresponding Base64 ciphertext entries in logs/room_1.log.
  3. Create a new room, jump between rooms, and point out the automatic join/leave events.
  4. Capture packets with Wireshark or tcpdump to underline that only ciphertext travels over the wire.
  5. Stop the relay; observe that room logs vanish while server.log persists with clean separators.
